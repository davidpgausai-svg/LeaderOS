### Phase 1: The Visual & UX Blueprint

Before coding, you must establish the visual hierarchy. Standard Gantt charts usually only have "Project" and "Task." Your requirement adds a "Priority" layer on top.

[Image of modern software gantt chart UI with nested rows and dependency lines]

**The Visual Specification:**

  * **Tier 1 (Priority):** A bold, distinct colored bar (spanning the total time of all projects beneath it). It acts as a collapsible container.
  * **Tier 2 (Project):** Indented beneath Priority. Represents specific initiatives.
  * **Tier 3 (Actions):** The actual executable bars. These are the leaves of the tree.
  * **Timeline:** A sticky header at the top (Months/Weeks).
  * **Interactivity:** Clicking a Priority should collapse/expand the Projects. Hovering over an Action shows the owner and status.

  const [viewMode, setViewMode] = useState<ViewMode>(ViewMode.Month);

### Why this works for "Strategy" specifically:

1.  **Nesting (`project` prop):** The key in the code above is assigning the `project` ID of the *child* to match the `id` of the *parent*. The library reads this and creates the collapsible arrow UI automatically.
2.  **Visual Distinction:** I added a `styles` object to the data. This visually separates the high-level **Strategy (Dark Colors)** from the actionable **Tasks (Light Colors)**, allowing executives to scan the big picture while managers track the details.
